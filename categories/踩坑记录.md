# 踩坑记录

## OpenClaw Gateway 外网访问配置 (2026-03-07)

### 问题描述

从 Windows 外网访问运行在 Ubuntu 虚拟机上的 OpenClaw Control UI。

### 环境信息

- **虚拟机 IP**: 192.168.222.128
- **公网 IP**: 120.226.236.68
- **Gateway 端口**: 18789
- **Gateway 监听**: 0.0.0.0:18789
- **防火墙**: ufw inactive（未启用）
- **SSH 服务**: 需要安装

### 踩坑过程

#### 踩坑 1: HTTP 访问被拒绝

**错误信息:**
```
control ui requires device identity (use HTTPS or localhost secure context)
```

**原因:**
浏览器安全策略，Control UI 要求安全上下文
- ✅ `https://`（任何地址）
- ✅ `http://localhost` 或 `http://127.0.0.1`（仅本地）
- ❌ `http://192.168.222.128`（普通 HTTP IP 访问被拒绝）

**解决方案:**
使用 SSH 隧道将远程地址映射到 localhost

#### 踩坑 2: Token 找错

**错误信息:**
```
unauthorized: gateway token missing
unauthorized: too many failed authentication attempts
```

**原因:**
使用了 device token（`~/.openclaw/identity/device-auth.json`），应该用 gateway token

**解决方案:**
正确的 token 在 `~/.openclaw/openclaw.json`：
```json
{
  "gateway": {
    "auth": {
      "token": "42cd3c837a572465ce69a05b0303d94fbdb1a0cd9903cb34"
    }
  }
}
```

### 完整解决方案

#### 步骤 1: 安装并启动 SSH 服务器

```bash
# 安装 OpenSSH Server
sudo apt update
sudo apt install -y openssh-server

# 启动并启用 SSH 服务
sudo systemctl start ssh
sudo systemctl enable ssh

# 检查 SSH 状态
ss -tulpn | grep :22
```

#### 步骤 2: 在 Windows 建立 SSH 隧道

打开 PowerShell 或 CMD：

```powershell
ssh -L 18789:127.0.0.1:18789 amor@192.168.222.128
```

**参数说明:**
- `-L 18789:127.0.0.1:18789`：本地端口转发
  - Windows 本地端口 `18789`
  - 虚拟机内部 `127.0.0.1:18789`
- `amor@192.168.222.128`：虚拟机用户名和 IP

**静默模式（不进入 shell）:**
```powershell
ssh -N -L 18789:127.0.0.1:18789 amor@192.168.222.128
```

#### 步骤 3: 访问控制 UI

保持 SSH 连接打开，在 Windows 浏览器访问：

```
http://127.0.0.1:18789?token=42cd3c837a572465ce69a05b0303d94fbdb1a0cd9903cb34
```

### 关键配置文件

**Gateway 配置** (`~/.openclaw/openclaw.json`):

```json
{
  "gateway": {
    "port": 18789,
    "mode": "local",
    "bind": "lan",
    "auth": {
      "mode": "token",
      "token": "42cd3c837a572465ce69a05b0303d94fbdb1a0cd9903cb34"
    },
    "controlUi": {
      "allowedOrigins": [
        "http://192.168.222.1:*",
        "http://192.168.222.128:18789",
        "http://localhost:18789",
        "http://127.0.0.1:18789"
      ]
    }
  }
}
```

**Token 位置:**
- ✅ **正确**: `~/.openclaw/openclaw.json` → `gateway.auth.token`
- ❌ **错误**: `~/.openclaw/identity/device-auth.json`（device token）

### 重要提示

1. **SSH 隧道连接需要一直保持打开**
   - 关闭 SSH 连接，隧道就断了
   - 建议使用 `ssh -N` 静默模式

2. **认证锁定**
   - 多次认证失败会触发临时锁定
   - 错误: `unauthorized: too many failed authentication attempts (retry later)`
   - 解决：等待 5-10 分钟，或重启 Gateway
   - 重启命令: `kill $(pgrep openclaw-gateway) && openclaw gateway start`

3. **网络连通性**
   - 确保虚拟机和 Windows 在同一网络
   - 测试连接: `ping 192.168.222.128`

4. **安全考虑**
   - Token 应该保密
   - 不要在公开场合分享 token
   - 可以定期轮换 token

### 故障排查

**问题: SSH 连接失败**
```bash
# 检查 SSH 服务状态
systemctl status ssh

# 检查端口监听
ss -tulpn | grep :22

# 检查防火墙
sudo ufw status
```

**问题: 浏览器仍然显示认证错误**
- 检查 token 是否正确
- 等待 5-10 分钟后重试（临时锁定）
- 重启 Gateway 清除失败记录

**问题: 隧道建立成功但无法访问**
- 检查 Gateway 进程是否运行: `ps aux | grep openclaw-gateway`
- 检查端口监听: `ss -tulpn | grep 18789`
- 检查日志: `openclaw gateway logs`

### 相关命令速查

```bash
# Gateway 操作
openclaw gateway start

openclaw gateway status
openclaw gateway logs --lines=50

# SSH 服务
sudo systemctl start ssh

# 网络检查
ss -tulpn | grep -E "22|18789"
lsof -i :18789 -P -n
```

访问 `http://127.0.0.1:18789` 应该能看到 OpenClaw Control UI 界面，并能正常使用所有功能。
