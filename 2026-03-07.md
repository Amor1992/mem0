## 网络配置踩坑记录

### 问题：从 Windows 外网访问 OpenClaw Control UI

**初始情况：**
- 虚拟机（Ubuntu）IP: 192.168.222.128
- 公网 IP: 120.226.236.68
- Gateway 监听: 0.0.0.0:18789
- 从虚拟机内部访问正常（HTTP 200）
- 从 Windows 访问超时

### 踩坑过程

#### 踩坑 1：HTTP 访问被拒绝
**错误：** `control ui requires device identity (use HTTPS or localhost secure context)`

**原因：** 浏览器安全策略，控制 UI 要求安全上下文
- ✅ `https://`（任何地址）
- ✅ `http://localhost` 或 `http://127.0.0.1`（仅本地）
- ❌ `http://192.168.222.128`（普通 HTTP IP 访问被拒绝）

#### 踩坑 2：Token 找错
**错误：** `unauthorized: gateway token missing` / `unauthorized: too many failed authentication attempts`

**原因：** 用了 device token 而不是 gateway token

**解决：**
- 正确 Token: `42cd3c837a572465ce69a05b0303d94fbdb1a0cd9903cb34`
- 位置: `~/.openclaw/openclaw.json` → `gateway.auth.token`

### 最终解决方案：SSH 隧道

#### 步骤 1：安装并启动 SSH 服务器

```bash
# 安装 OpenSSH Server
sudo apt install -y openssh-server

# 启动并启用 SSH 服务
sudo systemctl start ssh
sudo systemctl enable ssh
```

#### 步骤 2：在 Windows 建立 SSH 隧道

打开 PowerShell 或 CMD：

```powershell
ssh -L 18789:127.0.0.1:18789 amor@192.168.222.128
```

参数说明：
- `-L 18789:127.0.0.1:18789`：本地端口转发
  - Windows 本地端口 18789
  - 虚拟机内部 127.0.0.1:18789
- `amor@192.168.222.128`：虚拟机用户名和 IP

#### 步骤 3：访问控制 UI

保持 SSH 连接打开，在 Windows 浏览器访问：

```
http://127.0.0.1:18789?token=42cd3c837a572465ce69a05b0303d94fbdb1a0cd9903cb34
```

### 关键配置文件

**Gateway 配置** (`~/.openclaw/openclaw.json`)：

```json
{
  "gateway": {
    "port": 18789,
    "mode": "local",
    "bind": "lan",
    "auth": {
      "mode": "token",
      "token": "42cd3c837a572465ce69a05b0303d94fbdb1a0cd9903cb34"
    },
    "controlUi": {
      "allowedOrigins": [
        "http://192.168.222.1:*",
        "http://192.168.222.128:18789",
        "http://localhost:18789",
        "http://127.0.0.1:18789"
      ]
    }
  }
}
```

### 提示

- SSH 连接需要一直保持打开
- 关闭 SSH 连接，隧道就断了
- 可以用 `-N` 参数避免进入 shell：`ssh -N -L 18789:127.0.0.1:18789 amor@192.168.222.128`
- "too many failed authentication attempts" 表示临时锁定，等待 5-10 分钟或重启 Gateway
