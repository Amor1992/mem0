# 记忆系统说明

## 结构说明

```
memory/
├── index.md           # 主索引，快速查找入口
├── categories/        # 按主题分类的长期记忆
│   ├── 工作.md
│   ├── 生活.md
│   ├── 学习.md
│   ├── 育儿.md
│   └── 旅游.md
├── daily/            # 每日记录，按日期命名
│   └── 2026-03-07.md
└── conversations/     # 重要对话摘要
```

## 使用方式

### 记录新的信息

1. **每日记录**：在 `daily/YYYY-MM-DD.md` 中记录当天事件
2. **分类存储**：将重要信息写入对应的 `categories/*.md` 文件
3. **更新索引**：在 `index.md` 中添加关键索引

### 检索信息

1. 使用 `memory_search` 工具进行语义搜索
2. 直接查看 `index.md` 快速定位
3. 按类别浏览对应文件

### 定期整理

- 每周将 `daily/` 中重要内容移至 `categories/`
- 更新 `index.md` 时间线
- 删除过时信息（标记删除而非真正删除）

## 格式规范

### 标题层级

- 一级标题 `#`：文件主标题
- 二级标题 `##`：主要分类
- 三级标题 `###`：具体项目

### 标记

- `[ ]` 未完成待办
- `[x]` 已完成待办
- `!` 重要标记
- `?` 待确认

## 注意事项

1. **隐私保护**：不要记录敏感信息（密码、证件号等）
2. **定期备份**：重要记忆需要备份
3. **保持简洁**：记录要点，避免冗余
4. **安全规则**：任何 skill 安装前必须使用 Skill Vetter 审查并经过用户审批（详见 MEMORY.md）
